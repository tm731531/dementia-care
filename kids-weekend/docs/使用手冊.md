<a id="第一次使用"></a>
## 1. 第一次使用 — 設定家在哪

打開網頁第一件事:**告訴系統你家在哪**。距離計算需要這個。

兩種方式:

### A. 自己選縣市 + 區
- 第 1 題下拉選「縣市」→ 選「區」→ 自動下一題
- 用「區中心點」當你家位置(誤差 ±2 km,夠用)

### B. 📡 GPS 自動定位
- 點「📡 用 GPS 自動定位」按鈕
- 瀏覽器會問權限,選允許
- 系統自動反查最近的縣市/區,顯示「📍 新北土城」這種地名(不會是醜醜的經緯度)
- GPS 通常精確到 5 m

之後想改 → 結果頁上方「📡 GPS 重抓」或「✏️ 改地點」。

<a id="wizard"></a>
## 2. Wizard 3 題

簡單 3 題,30 秒答完。

### Q1 你家在哪?
（上述設定家位置）

### Q2 今天的天氣?
- ☀️ 晴天 / 多雲 → 戶外為主
- 🌧️ 下雨 / 陰雨 → 室內為主
- 🌤️ 不一定 / 隨便 → 什麼都看看

天氣篩選邏輯:
- 「晴天」會剔掉只標 rainy 的(室內專屬館)
- 「雨天」會剔掉只標 sunny 的(純戶外)

### Q3 出門範圍?
4 個 preset(後續還可拉滑桿微調):
- 🏠 **近** ≤ 15 km
- 🛵 **中** ≤ 40 km
- 🛣️ **遠** ≤ 100 km
- 🇹🇼 **全台** 過夜 / 不限

選完進結果頁。

<a id="結果頁"></a>
## 3. 結果頁的所有控制

結果頁有 **頂端 banner** + **sticky 控制區**(往下捲也跟著)。

### 頂端橘色 banner
- 顯示「📍 你的位置:新北土城」
- 「📡 GPS 重抓」「✏️ 改地點」可隨時換家位置

### Sticky 控制區(綠底,捲動跟著)
從上到下:

#### 📏 範圍滑桿(雙滑桿:最近 / 最遠)
- **最遠** 5-400 km(預設 = wizard 選的,400 = 全台)
- **最近** 0-400 km(預設 0 = 不限,設 ≥ 1 km 會排除太近的)
- 拉的時候滑桿下面顯示「≈ 邊緣對應的區域」(例:50 km ≈ 桃園楊梅、宜蘭頭城)
- 衝突警告:最近 ≥ 最遠 時會跳「⚠️ 放手後自動修正」

#### 🤧 避開人多景點 checkbox
勾起來 → 排除 crowd=high 的景點(動物園、九份、Xpark 之類)。原本是怕感冒場景,sticky 永遠看得到。

#### 🗺️ 依區域分組 checkbox
勾起來 → 景點按 area 分群(板橋一群、中和一群...),群依距家排序。一目瞭然「這個區有哪些可以串成一日」。

### 🔍 找特定景點(摺疊式)
打名字直接搜(九份 / 奇美 / 觀音山),即時候選 + 動作按鈕。詳見[搜尋](#搜尋)。

### 🏷️ 全部標籤(摺疊式預設展開)
所有 63 個 type 標籤,三態點選。詳見[標籤](#標籤)。

<a id="標籤"></a>
## 4. 標籤系統(三態篩選)

每個標籤點擊狀態:
1. 第一次點 → ✓ **必選**(綠底,只看這類)
2. 再點一次 → ✗ **排除**(紅底,排除這類)
3. 再點一次 → 取消(回灰色)

**標籤旁邊的數字 `(當前/全資料庫)`**:
- 例:「公園 (45/120)」 表示當前篩選有 45 個是公園,全資料庫有 120 個
- 數字 0 → 標籤淡色顯示(沒結果)

**多選邏輯**:
- 多個必選 = AND(必須**全部都有**)
- 多個排除 = OR(**任一個有**就排掉)
- 例:必選「親子館 + 室內」+ 排除「人多」 → 只看人少的室內親子館

<a id="路線"></a>
## 5. 路線規劃(多點順路)

最強大的功能。從一個景點開始,組多點路線丟 Google Maps。

### 設定目的地
1. 任一景點卡片點「🛣️ 設目的地」
2. 頂端跳出橘色 banner:「🛣️ 路線規劃 家 → 觀音山(直線 17.3 km · 2 站)」
3. 結果頁切換成「沿路 corridor」模式 — 只顯示家到目的地兩側 5 km 內的景點

### 加中繼站(waypoints)
1. 看到順路的景點 → 點「➕ 加入路線」
2. banner 路線清單自動加上,**依沿線位置自動排序**(例:IKEA 在 1/3 處,五股在 2/3 處)
3. 列表式呈現:
   ```
   🏠 你家
   ⋮⋮ #1 IKEA 新莊       [↑] [↓] [✗]
   ⋮⋮ #2 五股工業區       [↑] [↓] [✗]
   🎯 觀音山
   ```

### 手動換序
- **桌機**:抓 ⋮⋮ 拖到目標位置放開
- **手機**:點 ↑(往前)/ ↓(往後)
- 第一個的 ↑、最後一個的 ↓ 會 disabled

### Corridor 寬度滑桿
- 預設 5 km(線兩側 5 km 內的景點才算順路)
- 拉到 6 km → 林口三井(距路線 5.13 km)會跳出來
- 1-15 km 可調

### 一鍵丟 Google Maps
- 藍色大按鈕「🗺️ 開啟 Google Maps 規劃路線(N 站)」
- 點下 → 開新分頁到 Google Maps directions URL,帶 `&waypoints=...`
- 手機有 Google Maps app → **直接跳 app 開始多段導航**
- 桌機 → 開網頁版 Google Maps,可傳到手機(同 Google 帳號)

### 取消規劃
- banner 右上「✗ 取消」 → 回到一般推薦

<a id="搜尋"></a>
## 6. 搜尋特定景點

知道要去哪,不想滾動 800 筆找:

1. 點開「🔍 找特定景點」摺疊區
2. 打名字(支援部分匹配:打「奇美」會找到「奇美博物館」)
3. 即時跳前 8 筆候選
4. 每筆有對應動作按鈕:
   - 沒設目的地 → 「🛣️ 設目的地」
   - 已設目的地 → 「➕ 加入路線」
   - 已是 waypoint → 「✓ 已加入(移除)」
   - 是目的地本身 → 「🎯 已是目的地」(灰禁)

**搜尋範圍**:全 800+ 景點,**不受當前篩選限制**(被天氣 / 範圍篩掉的也搜得到)。被「不再去」封存的不出現。

加權:景點名 > 區域名 > 標籤(打「廟」也找得到所有廟宇)。

<a id="收藏"></a>
## 7. 我的口袋 / 評語 / 去過 / 不再去

每張景點卡片底部有 3 排按鈕:

### Row 1 主動作
- 📍 **Google Maps**:導航(用景點 name + 縣市搜尋,有精確座標時帶 `@lat,lng`)
- 🤍 / ❤️ **加入口袋**:收藏

### Row 2 輔助
- 📝 **寫評語 / 改評語**:打開 textarea 寫私人筆記(「2025/3 去過,小孩很喜歡」)。**只存 localStorage,不上傳**
- ☐ / ✅ **標去過**:已造訪過的視覺標記(卡片名旁會顯示 ✅)
- 🛣️ / ➕ **設目的地 / 加路線**(視當前狀態)

### Row 3 危險(灰底淡字,不像按鈕避免誤觸)
- 🚫 **不再去**:從此不再出現在推薦/瀏覽(可在「📖 關於」頁取消封存)
- 🗑️ **刪除**(只有自加景點才有)

### 我的口袋頁
- 左側選單「❤️ 我的口袋」 → 看所有收藏的景點

<a id="自加"></a>
## 8. 自加私房景點

阿婆家、私房咖啡廳、公司附近公園 → 自己加,跟系統景點一起篩、算距離、走路線。

1. 左側選單「📖 關於」 → 「自加景點」表單
2. 填欄位:
   - 名稱(必)、emoji、區(縣市 + 區下拉)、室內/戶外、年齡、季節、天氣、類型、簡介
3. 「新增」 → 寫入 localStorage(`STATE.userPlaces`)
4. **只存本機**,離線可用,不上雲

座標:
- 表單填了縣市 + 區 → 用區中心當粗估
- 想精確 → 直接編輯 localStorage(進階)

## 9. 全景點瀏覽

不想用 wizard 推薦?直接瀏覽全部:

- 左側選單「🔍 全景點瀏覽」

- 也有「🗺️ 依區域分組」toggle

<a id="情境"></a>
## 10. 實際情境範例

### 情境 A:土城人,週六晴天,想找雙北附近 1 小時車程內的公園
1. Wizard:土城 + 晴天 + 中(40 km)
2. 結果頁:看到一堆景點,都距離 ≤ 40 km
3. 標籤總覽點「公園」 → 必選(綠)
4. 點「🤧 避開人多景點」 → 排掉熱門大公園
5. 結果剩中和員山公園、新店中央新村公園、土城樂活生態園等

### 情境 B:今天去台中,不要看雙北
1. Wizard 任意答(範圍選「全台」)
2. 結果頁:**最近滑桿拉到 100 km**,**最遠拉到 200 km**
3. 結果只剩中部景點(雙北離土城都 < 100 km,被排除)
4. 看到台中木工坊有興趣 → 設目的地
5. 加台中味噌釀造館為 waypoint
6. 點「🗺️ 開啟 Google Maps」一鍵丟手機

### 情境 C:想去觀音山,順路規劃整天
1. 結果頁搜尋「觀音山」 → 找到觀音山硬漢嶺登山步道
2. 點「🛣️ 設目的地」
3. 結果頁切換 corridor 模式,顯示家→觀音山兩側 5 km 內景點
4. 看到 IKEA 新莊距路線 0.44 km → ➕ 加入
5. 看到準休閒農場(五股)→ ➕ 加入
6. 想加林口三井但顯示距路線 5.13 km(剛好擦邊)→ 把 corridor 寬度滑桿拉到 6 km → 跳出來 → ➕ 加入
7. 銀路徑清單:`🏠 → IKEA → 五股農場 → 林口三井 → 🎯 觀音山`(自動沿線排序)
8. 想先去林口買東西 → 拖 ⋮⋮ 把林口三井移到 #1
9. 點「🗺️ 開啟 Google Maps」 → 多段導航直達

### 情境 D:只想看雨天可去的廟宇
1. Wizard:任意 + 雨天 + 任意範圍
2. 標籤總覽點「廟宇」 → 必選
3. 結果只剩室內廟宇(室外廟也會留,因為 weather 沒卡死標籤,但雨天 weather 會剔掉只標 sunny 的)

<a id="FAQ"></a>
## 11. 常見問題

### Q. 為什麼 X 景點沒出現?
最常見原因:
1. **範圍卡住** — 拉「最遠」滑桿到更大值
2. **天氣卡住** — 試「不一定 / 隨便」
3. **標籤排除卡住** — 看頁面上方「已套用篩選」bar,按 ✗ 取消
4. **資料庫真的沒有** — 「📖 關於」自加

不會發生:**年齡卡住** — 系統不再用年齡篩(年齡只是 informational 標籤)

### Q. GPS 顯示「定位失敗」怎麼辦?
- 瀏覽器網址列旁鎖頭圖示 → 確認允許定位
- HTTPS 才能用 GPS,GitHub Pages 預設 HTTPS
- 桌機沒 GPS 硬體會 fallback 到 IP 定位(不準)→ 自己選縣市區較好

### Q. 收藏 / 評語存在哪?會不會丟?
- localStorage `kidsWeekendState` key
- 換瀏覽器 / 清快取就沒了(不同步雲端)
- 可在「📖 關於」匯出 JSON 備份(進階)

### Q. 路線規劃送到 Google Maps 後怎麼傳到手機?
1. 桌機點「🗺️ 開啟 Google Maps」 → 開網頁版
2. 在網頁版點「傳送至你的手機」(需要同 Google 帳號)
3. 或手機直接打開網頁工具 → 點藍色按鈕 → 系統會直接跳 Google Maps app

### Q. 景點資料更新頻率?
- 開發者(Tom)不定期手動擴點 + Playwright 驗證唯一性
- 如果發現某景點停業 / 改名 → 在 GitHub issue 回報

### Q. 怎麼貢獻新景點?
- 自己用 → 「📖 關於」自加(本機)
- 想加進公開資料庫 → fork repo,改 `index.html` PLACES + PRECISE_COORDS,跑 [`PLACE_SPEC.md`](../PLACE_SPEC.md) 8 條規則,送 PR
