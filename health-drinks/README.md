# 健康飲品成分分析 — 醫療營養品比較網站

> 照顧失智媽媽要買醫療營養品（安素、葡勝納 SR、倍速益、補體素、倍力素），每瓶成分表細節不一樣，貨架前肉眼比不出來。
> 所以做一個純前端比較網站。在家樂福、大樹連鎖藥局前掏手機看一下就能挑。

資料是**拍包裝標籤 → AI 辨識擷取**手動進 JSON，不是爬蟲。

🔗 **網站**：<https://tm731531.github.io/dementia-care/health-drinks/docs/>

---

## 為什麼做這個

失智長輩需要營養補充——有的要高蛋白（肌少症）、有的要低糖（糖尿病）、有的要高熱量（食慾差體重掉）、有的要好吸收膳食纖維。市售醫療營養品：

- **亞培安素** 系列（含 HMB、葡勝納 SR 糖尿病配方、倍力素癌症病人配方…）
- **費森尤斯卡比**（倍速益）
- **補體素**（蛋白質補充）
- **三多補體健**
- 加上嬰幼兒配方奶粉、各式高蛋白粉、雞精

每瓶包裝側邊印一堆密密麻麻的數字，光是「熱量多少」好比，但**微量元素配方差異**（維生素 B12、碘、硒、鉻）才是營養師會關心的——貨架前根本比不出來。

市售比價 app：
- 只給售價，不給營養
- 塞廣告、要登入
- 資料涵蓋不到台灣的醫療營養品品牌

所以自己做一個自用的：
- **拍**包裝成分標籤
- **AI 辨識**（Claude / Gemini）擷取成結構化 JSON
- **全欄位填**（含微量元素，不省略）
- 純靜態網站部署 GitHub Pages
- 比較矩陣欄位**動態生成**：某筆有填碘，那一列才出現

### 新增一筆資料的流程

這是本專案**最重要的工作流**，brain file `~/.claude/projects/-home-tom/memory/brain/health-drinks-data-entry.md` 有完整規則：

1. 買到一瓶新的（或找到有興趣的產品包裝）
2. **拍下正反面**成分標籤（重點是營養成分表）
3. 丟給 Claude/Gemini 辨識，要求「全填包含微量元素，不省略」
4. AI 產出一個符合 `drinks.js` schema 的 entry：
   ```js
   {
     id: "abbott-ensure-hmb-original",
     name: "安素 HMB 升級配方（原味）",
     brand: "亞培 Abbott",
     volume_ml: 237,            // 每份(沖泡後)體積,粉要換算成沖調後體積
     image: "images/亞培安素HMB.jpg",
     categories: ["醫療營養品"],
     nutrients: {
       "熱量(kcal)": 262,
       "蛋白質(g)": 11,
       "糖(g)": 14,
       "鈉(mg)": 150,
       // ... 標籤有什麼就填什麼
       "維生素A(mcgRE)": 200,   // 命名規範見 brain file
       "碘(mcg)": 30,
       "硒(mcg)": 14,
       // 微量元素全填
     }
   }
   ```
5. 把圖片放 `docs/images/`，entry 放進 `docs/data/drinks.js` 的 `DRINKS` 陣列
6. 瀏覽器打開網站確認新品項有出現在表格、比較矩陣

---

## 資料以「每份」儲存，前端換算 per 100ml

**重要**：`nutrients` 欄位值是**每份**（例 237ml 安素 = 262 kcal），不是 per 100ml。網站透過 `volume_ml` 做換算讓不同容量的產品公平比較。

粉狀產品的 `volume_ml` 要填**沖調後**體積（例 25g 粉 + 200ml 水 → `volume_ml: 225`）。

---

## 功能

- **排行表格**：依熱量、蛋白質、糖、鈉、鈣…排序
- **每 100ml 換算**：公平比較不同容量的產品
- **比較矩陣**：選幾瓶並排看差異，格子自動顏色標記（綠=優、紅=劣），左欄項目動態決定
- **圖片對照**：拍下來的產品包裝照
- **搜尋/篩選**：按品牌、分類

## 關鍵設計原則（來自 brain file）

1. **「加進去」= 全填含微量元素**：使用者叫 AI 補新資料時，只填巨量營養素會讓比較矩陣欄位缺失
2. **每份儲存，不是 per 100ml**：前端換算，否則雙重換算會錯
3. **粉狀 `volume_ml` 是沖泡後體積**：不是粉末重量
4. **欄位命名嚴格**：`熱量(kcal)` / `蛋白質(g)` / `鈉(mg)` / `碘(mcg)` / `維生素A(mcgRE)` / `菸鹼素(mgNE)`
