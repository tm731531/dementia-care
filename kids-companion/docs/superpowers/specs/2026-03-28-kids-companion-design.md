# 小朋友學習樂園 — 設計規格

## 概述

給 3–6 歲兒童在平板使用的學習互動 app。寓教於樂，親子兩用，中英文可切換。
所有功能在單一 `index.html`，完全離線可用。

---

## 兒童友善設計原則

### 配色 — 護眼原則
- **禁用高飽和霓虹色**（如純 `#FF0000`、`#00FF00`）：改用柔和版本（`#E8724A`、`#98D8C8`）
- **背景不用純白**：用米白 `#FFF8F0`，減少螢幕藍光對比刺激
- **按鈕底色用暖色系**，避免冷白 + 高飽和色組合
- **答錯反饋不用紅色閃爍**：改用橘色輕微搖晃動畫
- **禁止快速閃爍**：任何動畫頻率不得超過每秒 3 次（避免光敏癲癇風險）

### 聲光效果 — 避免恐懼
- **禁止突然大音量**：所有音效統一音量，不可突然爆音
- **錯誤提示音**：使用低沉溫和的「啵」聲，不用尖銳蜂鳴
- **答對音效**：輕快短促（0.3–0.5 秒），不拉長或過於誇張
- **慶祝動畫**：五彩紙屑從上方緩緩飄落（非爆炸式），持續 2–3 秒
- **不使用**：倒數計時音、警報聲、角色哭泣動畫、紅色大叉叉
- **答錯語氣**：「哎呀，再試一次！」不出現「錯了！」「不對！」
- **禁用黑暗恐怖元素**：任何故事或活動不出現骷髏、怪物、黑暗場景

### 語音 — 順暢性
- **發音前取消前一句**：`speechSynthesis.cancel()` 後再 `speak()`，避免聲音疊加
- **語句間自然停頓**：選項間隔 600ms，讓孩子有時間處理
- **朗讀前清除 emoji 和標點**：避免念出「emoji 名稱」或標點符號
- **語速 0.85，音調 1.2**：稍高音調讓孩子更容易集中注意力
- **長句拆分**：超過 15 個字拆成兩次朗讀，中間停頓 400ms
- **語言切換即時生效**：切換後下一句即用新語言
- **iOS 限制**：iOS 需使用者互動才能啟動語音，第一次進活動提示點畫面

## 視覺風格

**溫暖自然**：溫柔不過度刺激，像繪本的感覺。

| 用途 | 色碼 |
|------|------|
| 主色（橘） | `#E8724A` |
| 背景（米白） | `#FFF8F0` |
| 輔色（草綠） | `#98D8C8` |
| 暖黃 | `#FFB347` |
| 卡片白 | `#FFFFFF` |
| 深字色 | `#333333` |
| 淺字色 | `#888888` |

- 圓角：20px（卡片）、50px（按鈕）
- 陰影：`0 4px 16px rgba(0,0,0,0.08)`
- 字型：Nunito → 蘋方 → 微軟正黑體（圓體優先）
- 基礎字級：22px；按鈕：28px；遊戲目標字：80–120px

## 年齡組設計原則

依台灣幼兒園分班制度，分為四個等級：

| 等級 | 年齡 | ageGroup 值 |
|------|------|-------------|
| 幼幼班 | 約 2–3 歲 | `toddler` |
| 小班 | 約 3–4 歲 | `small` |
| 中班 | 約 4–5 歲 | `middle` |
| 大班 | 約 5–6 歲 | `large` |

### 各等級行為差異

| | 幼幼班 | 小班 | 中班 | 大班 |
|--|--------|------|------|------|
| 選項數 | 2 個 | 2–3 個 | 3–4 個 | 4–6 個 |
| 目標字大小 | 140px | 120px | 100px | 80px |
| 語音行為 | 朗讀題目＋所有選項，每題重複一次 | 朗讀題目＋所有選項 | 朗讀題目，hover 念選項 | 只朗讀題目 |
| 答錯處理 | 直接高亮正確答案 | 高亮正確答案 | 提示再試，第2次才高亮 | 只提示再試 |
| 單局輪數 | 4 輪 | 5 輪 | 6 輪 | 8 輪 |
| 拖拉支援 | 否（只點選） | 否 | 是 | 是 |

## 活動規格

### 1. 認中文字（page-chinese）
| 等級 | 內容 |
|------|------|
| 幼幼班 | 2 選項，圖畫字（象形字：山🏔️、日☀️、月🌙、水💧） |
| 小班 | 2–3 選項，常見筆畫少的字（大、小、上、下、手、口） |
| 中班 | 3–4 選項，國小預備字彙，加入同部首字 |
| 大班 | 4–6 選項，形近字干擾（形/邢/刑/型） |

### 2. 英文字母（page-abc）
| 等級 | 內容 |
|------|------|
| 幼幼班 | 認 A–E，超大字母配動物圖，2 選項 |
| 小班 | 認 A–J，字母配圖，3 選項 |
| 中班 | 認 A–Z，4 選項，配單字發音（A is for Apple） |
| 大班 | 認 A–Z，4 選項，加入大小寫配對 |

### 3. 數數（page-counting）
| 等級 | 內容 |
|------|------|
| 幼幼班 | 1–3 個物品，2 選項 |
| 小班 | 1–5 個物品，3 選項 |
| 中班 | 1–10 個，4 選項 |
| 大班 | 1–20，5 選項；進階有簡單加法（2+3=?） |

### 4. 詞彙學習（page-vocab）
| 等級 | 內容 |
|------|------|
| 幼幼班 | 2 選項，日常最基本物品（奶瓶、球、床） |
| 小班 | 3 選項，常見物品（蘋果、狗、車子） |
| 中班 | 4 選項，中等常見（草莓、公車、雨傘） |
| 大班 | 6 選項，較少見（企鵝、飛機、望遠鏡） |

### 5. 顏色形狀（page-colors-shapes）
| 等級 | 內容 |
|------|------|
| 幼幼班 | 3 種基本色（紅黃藍），2 選項，圓形/方形 |
| 小班 | 4 種基本色，3 種形狀，高對比 |
| 中班 | 6 種顏色，4 種形狀，中對比 |
| 大班 | 8 種顏色（含相近色），6 種形狀 |

### 6. 音樂節奏（page-music）
| 等級 | 內容 |
|------|------|
| 幼幼班 | 點大型 emoji 樂器，自由發出音效，純探索 |
| 小班 | 同上，加入「跟我一樣」模仿單個音效 |
| 中班 | 聽 2–3 個音的節奏樣式，照樣打拍子 |
| 大班 | 聽 3–5 個音的節奏樣式（Simon Says） |

### 7. 拼圖（page-puzzle）
| 等級 | 內容 |
|------|------|
| 幼幼班 | 2×2（4 塊），點選放回（非拖拉） |
| 小班 | 2×2（4 塊），拖拉 |
| 中班 | 2×3（6 塊），拖拉 |
| 大班 | 3×3（9 塊），拖拉 |

### 8. 記憶翻牌（page-memory）
| 等級 | 內容 |
|------|------|
| 幼幼班 | 2×2（4 張，2 對） |
| 小班 | 2×3（6 張，3 對） |
| 中班 | 3×3（9 張，配對 +1 張圖案提示） |
| 大班 | 3×4（12 張，6 對） |

### 9. 排序分類（page-sort）
| 等級 | 內容 |
|------|------|
| 幼幼班 | 大小排序 2 個（明顯差距），點選 |
| 小班 | 大小排序 3 個（差距大），點選 |
| 中班 | 大小排序 3–4 個，或 2 類別分類，拖拉 |
| 大班 | 4–5 個物品分入 3 類別，拖拉 |

### 10. 互動繪本（page-story）
- 書架頁顯示可選的故事（2×N 格）
- 每本書：多頁，每頁有背景圖（emoji 組合）、文字、語音
- 點擊畫面中的物件有聲音/動畫反應
- 左右滑動或點箭頭翻頁
- 內建故事資料格式：
```javascript
const STORIES = [
  {
    id: 'monkey-king',
    title: { zh: '孫悟空大鬧天宮', en: 'The Monkey King' },
    cover: '🐒',
    pages: [
      {
        bg: ['☁️','🏔️','🌳'],
        text: { zh: '很久很久以前，有一隻石頭裡跑出來的猴子...', en: 'Long long ago, a monkey was born from a magic stone...' },
        objects: [
          { emoji: '🐒', sound: '哇！我出生了！', x: 50, y: 60 }
        ]
      }
    ]
  }
];
```
- junior：語音自動朗讀每一頁文字
- senior：點播放鍵才朗讀

## 遊戲化系統

### 角色選擇（初次啟動）
- 全屏歡迎畫面，選一個角色：🐱 🐶 🐼 🦊 🐧 🐸
- 選定後角色出現在首頁 header，陪伴整個 app
- 之後可在設定中更換

### 星星系統
- 每完成一個活動得 1 顆星
- 首頁顯示今日進度（例：⭐⭐⭐☆☆）
- 集滿 5 顆：全屏慶祝動畫（五彩紙屑 + 角色跳舞）
- 每天重置

### 貼紙書
- 每個活動對應一張專屬貼紙
- 首次完成解鎖，點首頁 header 的 🎴 按鈕查看貼紙書頁
- 已解鎖：彩色顯示；未解鎖：灰色輪廓（共 10 格）

### 連續天數
- 比對 lastPlayDate 與今天日期
- 進 app 時顯示「你連續玩了 N 天！」
- 中斷後重置為 1

### 慶祝動畫
- 答對時：小型五彩紙屑從頂部飄落 + 角色跳一下
- 完成活動：大型慶祝動畫 + 星星飛進進度條
- 集滿 5 星：全屏派對動畫（3 秒）

### 家長給讚
- 遊戲中右下角小按鈕 👏（半透明，不干擾遊戲）
- 按下後：全屏出現「爸爸/媽媽說你好棒！❤️」3 秒

### 神秘寶箱
- 當天完成 3 個（以上）活動後，首頁出現寶箱動畫
- 點擊開箱：顯示「今日小知識」（一張圖 + 一句話 + 語音）
- 小知識內容：內嵌在 HTML 裡的 `TRIVIA` 陣列，例如：
  - 🦒「長頸鹿的脖子有 1.8 公尺！比你還高很多喔」
  - 🌙「月亮不會自己發光，它反射太陽的光」
  - 🐬「海豚睡覺時只睡一半的腦」
  - 🌱「向日葵白天會轉頭追著太陽走」

### 搖一搖
- 偵測 devicemotion（加速度 > 15）
- 觸發「今日驚喜」：隨機跳到一個活動，搭配動畫

## 使用時長提醒

依兒科建議，3–6 歲每次使用螢幕時間不超過 30–45 分鐘。

### 建議時長
| 年齡組 | 單次建議上限 |
|--------|-------------|
| junior（3–4 歲） | 30 分鐘 |
| senior（5–6 歲） | 45 分鐘 |

### 計時邏輯
- App 開啟時記錄 `sessionStart` 時間戳
- 每分鐘背景檢查已使用時間
- 今日累積時間存入 localStorage（`todayPlayMinutes`、`todayDate`），隔天重置

### 提醒機制（不強制中斷）
| 時間點 | 行為 |
|--------|------|
| 距離上限剩 5 分鐘 | 畫面頂部出現溫和橫幅：「再玩 5 分鐘就要讓眼睛休息一下喔 👀」 |
| 達到建議上限 | 全屏休息提示（角色動畫）+ 「眼睛累了，我們去喝水伸懶腰！」|
| 家長點「繼續玩」 | 關閉提示，延長 10 分鐘後再次提醒 |
| 家長點「好，休息」 | 回首頁，顯示今日完成成就 |

### 全域狀態補充
```javascript
// 加入 APP 物件
sessionStart: null,      // Date.now()，每次開啟 app 設定
todayPlayMinutes: 0,     // 今日累積分鐘數
todayDate: ''            // 'YYYY-MM-DD'，用於每日重置
```

### 設計原則
- **提醒不懲罰**：語氣溫柔，不出現「時間到！不能玩了」等強制語句
- **家長有最終決定權**：所有提示都有「繼續玩」選項
- 設定面板可關閉提醒（給有特殊需求的孩子）

## 資料轉移機制

localStorage 只存在當前裝置，換裝置即遺失。設定面板提供匯出/匯入功能讓家長轉移孩子的進度。

### 匯出
- 設定面板底部「匯出進度」按鈕
- 將完整 APP 狀態序列化為 JSON，觸發瀏覽器下載
- 檔名：`kids-companion-backup-YYYYMMDD.json`
- 格式：
```json
{
  "version": 1,
  "exportedAt": "2026-03-28T14:00:00",
  "data": {
    "language": "zh",
    "ageGroup": "small",
    "character": "🦊",
    "stars": 3,
    "stickers": ["page-chinese", "page-memory"],
    "streak": 5,
    "lastPlayDate": "2026-03-28",
    "completedToday": ["page-chinese"]
  }
}
```

### 匯入
- 設定面板底部「匯入進度」按鈕
- 點擊後開啟系統檔案選擇器（`<input type="file" accept=".json">`）
- 讀取 JSON，驗證 `version` 欄位存在且為數字
- 驗證通過：覆蓋 localStorage，顯示「進度已還原！」並重新載入
- 驗證失敗（格式錯誤）：顯示「檔案格式不對，請選正確的備份檔」

### 注意事項
- 匯入會**完全覆蓋**現有進度，操作前顯示確認提示
- 不支援跨版本合併，只做整包覆蓋
- 匯出/匯入操作均為純前端，不傳送任何資料到外部

## 陪伴者設計考量

此 app 的陪伴者不限於家長，設計時需考慮所有可能使用者：

| 陪伴者 | 使用情境 | 設計考量 |
|--------|----------|----------|
| 家長 | 家庭，每日使用 | 追蹤進度、習慣養成、換裝置備份 |
| 保母 | 家庭，替代照顧 | 介面直觀、不需背景知識即可操作 |
| 幼教員／教保員 | 托嬰中心、課後照顧 | 快速切換班別、一對多陪伴 |
| 幼兒園老師 | 教室，團體或個別 | 評量觀察用途，可搭配課程 |
| 兒童醫生 | 診間，發展評估 | 快速設定、觀察孩子反應、不需帳號 |

### 對應的設計調整

**1. 「陪伴者給讚」取代「家長給讚」**
- 設定中可選陪伴者稱謂：爸爸 / 媽媽 / 老師 / 阿姨 / 醫師 / 陪伴者
- 給讚時顯示：「[稱謂] 說你好棒！❤️」
- 預設值：「陪伴者」

**2. 介面自說明性（Self-explanatory）**
- 首頁設定流程不超過 2 步即可開始使用
- 班別選擇在首頁明顯呈現，任何陪伴者都能快速找到
- 不依賴記憶（每次進 app 都能看到當前設定）

**3. 無帳號設計**
- 不需登入，打開即用
- 適合診間臨時使用（兒童醫生評估）
- 匯出/匯入負責跨裝置資料轉移

**4. 使用時長彈性**
- 家庭情境：標準建議時長（30/45 分鐘）
- 教室/診間情境：陪伴者可隨時從設定關閉時長提醒

**5. 稱謂儲存在 localStorage**
```javascript
// APP 物件加入
companionTitle: '陪伴者'  // 顯示在給讚訊息
```
