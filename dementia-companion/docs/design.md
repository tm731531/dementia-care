# UI/UX 設計規格

## 設計理念
為失智症長輩打造一個「看得清楚、按得到、不會怕」的互動介面。
所有設計決策以「減少困惑、增加安心感」為核心。

---

## 配色方案（高對比模式）

| 用途 | 色碼 | 說明 |
|------|------|------|
| 背景色 | `#000000` | 純黑，最大對比 |
| 卡片背景 | `#FFFFFF` | 純白，與黑色背景形成極致反差 |
| 卡片上文字 | `#000000` | 純黑字在白卡上 |
| 背景上文字 | `#FFFFFF` | 白字在黑背景上 |
| 次要文字 | `#CCCCCC` | 淺灰，用於輔助資訊 |
| 主按鈕 | `#FF6600` | 橘色，吸引注意 |
| 正確回饋 | `#00AA00` / `#006600` | 綠色系 |
| 錯誤提示 | `#CC6600` | 橘色，溫和不刺激 |
| 重點文字 | `#CC0000` | 紅色，目標數字/文字 |
| 邊框 | `#CCCCCC` | 淺灰邊框區隔選項 |

### 配色原則
- 不使用近色系組合（如深藍配深紫）
- 文字與底色對比度 > 7:1（符合 WCAG AAA）
- 白色卡片上一律用黑色文字，黑色背景上用白色文字

## 遊戲顏色系統

### 形狀配對 & 認數字 — 白底紅字
- 數字選項：白底 `#FFFFFF`，紅字 `#CC0000`
- 形狀目標：紅色系色塊

### 顏色辨識 — 高對比不同色系
紅 `#FF0000`、藍 `#0000FF`、黃 `#FFD700`、綠 `#009900`、
橘 `#FF6600`、紫 `#7700CC`、粉紅 `#FF69B4`、咖啡 `#8B4513`、
黑 `#000000`、白 `#FFFFFF`、灰 `#888888`、天藍 `#00BFFF`

## 字體規格

- **字型優先順序**：微軟正黑體 → 蘋方 → Noto Sans TC → sans-serif
- **基礎字級**：20px
- **按鈕文字**：24px 以上
- **標題文字**：32px
- **遊戲目標字**：100px - 120px（特大，清楚可見）
- **遊戲選項字**：52px - 72px
- **行高**：1.6

## 按鈕規格

- 最小高度：64px
- 最小寬度：64px
- 圓角：16px
- 觸控區域：最小 48x48px
- 按鈕間距：至少 16px（避免誤觸）
- 按下效果：輕微縮放（scale 0.95）

## 語音回饋

- 使用 Web Speech API（`speechSynthesis`）
- 語言：`zh-TW`（繁體中文）
- 語速：0.7（慢速，適合長輩聽清楚）
- 語調：1.1（稍高，聽起來溫暖）
- 自動去除 emoji 後朗讀
- 鼓勵訊息顯示在頂部，不擋遊戲區域

### 難度與語音對應

| 難度 | 語音行為 |
|------|----------|
| 1–2（輕度） | 只朗讀題目 |
| 3–6（中度） | 朗讀題目；手指滑過/hover 選項時念出 |
| 7–8（重度） | 朗讀題目 + 所有選項（間隔 0.8 秒），重複一次 |

## 難度滑桿 UI

- 全寬滑桿，左標「輕度 (1-2)」、右標「重度 (7-8)」
- 中間顯示目前數值（輕度/中度/重度 + 數字）
- 顏色漸層：輕度端藍色 → 重度端紅色
- 拖動後放開才觸發重新抽選

## 動畫規格

| 動畫 | 時長 | 用途 |
|------|------|------|
| 鼓勵滑入 | 0.3s | 正確回饋從頂部滑入 |
| 鼓勵消失 | 1.5s | 回饋自動消失 |
| 搖晃 | 0.5s | 答錯提示 |
| 發光 | 0.5s | 正確選項高亮 |
| 頁面切換 | 0.3s | 淡入 |
