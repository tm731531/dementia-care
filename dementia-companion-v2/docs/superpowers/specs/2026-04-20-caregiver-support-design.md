# 照護者支援功能設計規格

日期：2026-04-20
版本：v2
狀態：Draft（待 review）

---

## 背景與動機

v2 目前的設計已經解決了「照護者每天要從 15 個遊戲裡挑一個」的決策疲勞（推薦式首頁）。但還有兩個尚未處理的照護者痛點：

1. **「我不知道跟長輩玩的時候要說什麼」**  
   照護者自己也累、也空白，即使選了活動，玩的過程中不知道怎麼開口、怎麼鼓勵。
2. **「家人問我今天爸爸怎麼樣，我講不出具體的」**  
   照護者完成了陪伴，但沒有一個輕鬆的方式把今天的狀態告訴其他家人。

這份 spec 針對這兩點設計兩個對應的小功能：**陪伴指南** 與 **今日摘要**。

## 範圍

### In scope
- **陪伴指南**：遊戲中給照護者的「可以說什麼」提示
- **今日摘要**：一次使用結束後，產出可貼到家族 LINE 的紀錄文字

### Out of scope
- 獨立的「話題卡」/「聊天提示」功能 → 屬於另一個 APP（下一個專案）
- 長期資料累積、趨勢圖、跨天紀錄 → 刻意不做
- 雲端同步、帳號系統 → 刻意不做
- 音樂 / 情緒開場 / 個人化資料（家族照片、生平）→ 刻意不做，未來可再評估

### 非功能性約束
- 維持 v2 單一 HTML 檔、純前端、離線可用
- **不引入任何資料庫**（IndexedDB 也不行）
- 唯一允許的 localStorage 新增 key：`elderName`（單一字串，僅用於摘要內稱呼）
- 所有活動紀錄、時長統計只保留在當次 session 的記憶體中，頁面 reload 即歸零

## 設計原則（核心）

整組功能的共同原則是**「可預期」**：

1. 照護者不需要解讀新的 UI，每次打開看到同樣位置、同樣格式、同樣模板
2. 提示與摘要都以「照護者能直接看到該說什麼」為最高目標——不是提醒「該做什麼」，是**給出可以直接對長輩說的完整句子**
3. 不讓照護者做決策（選模板、填空格、設心情），讓他專心陪長輩

這也是為什麼把「提示唸出聲」刻意排除——那是給長輩聽的指令，會搶走照護者用自己聲音、自己關係去講的空間。

## 功能 1：陪伴指南（遊戲中提示）

### 目的
在遊戲的三個關鍵時機，給照護者一句**可以直接對長輩說的話**。

### 資料結構

每個遊戲有 3 條提示，對應 3 個時機：

```js
TIPS = {
  color: {
    start:    '哇，好多顏色，你以前最愛穿什麼顏色？',
    correct:  '對！你好厲害',
    complete: '今天陪你看顏色，我也覺得很漂亮',
  },
  pairs: { start: '...', correct: '...', complete: '...' },
  // ... 15 個遊戲
};
```

提示內容全部是 **可直接對長輩說的完整句子**，不是操作說明（❌「引導他觸控卡片」），也不是照護小技巧（❌「給他多一點時間」）。

### UI 行為

```
遊戲畫面進入時
  ↓
頂部 banner fade in
  顯示 start 提示
  5 秒後 fade 成右上角小 💬 icon
  ↓
長輩答對時
  banner fade in
  顯示 correct 提示
  5 秒後 fade 成 💬
  ↓
遊戲完成時
  complete 提示直接顯示在 complete screen 上（不收起）
```

使用者可以**隨時點 💬 icon**重新展開當下時機的提示。  
（不提供「換一條」功能——一個時機就一個提示，避免做選擇。）

### Banner 樣式

- 位置：頂部，遊戲 header 下方、round-indicator 上方
- 背景：`#333333`（深灰，與 `#000` 純黑有清楚邊界，白字對比 12.6:1 達 AAA）
- 文字：白 `#FFFFFF`，`font-size: 18px`，`line-height: 1.5`
- 內距：16px 24px
- 圓角：12px
- 前綴圖示：💬（說的話）
- 格式：`💬 [完整一句話]`

### 💬 icon（摺疊狀態）

- 位置：遊戲畫面右上角，固定 `position: absolute; top: 16px; right: 16px`（在該 screen 內）
- 大小：44x44 圓形（符合 48px 觸控區域建議）
- 背景：`#333333` 實心（不用半透明，避免對比問題）
- border：`2px solid #FFFFFF`
- icon 本身：💬 22px
- hover：背景變 `#FFFFFF`，icon 顏色反轉（黑色不適用 emoji，改用提升 border 亮度即可）
- 點擊：重新展開目前時機的 banner（和 banner 一樣用 💬，保持視覺統一）

### 15 個遊戲的內容（完整列表）

| 遊戲 | start | correct | complete |
|------|-------|---------|----------|
| color | 哇，好多顏色，你以前最愛穿什麼顏色？ | 對！你好厲害 | 今天陪你看顏色，我也覺得很漂亮 |
| pairs | 這個要記一下，慢慢翻沒關係 | 哎喲！你記得耶 | 你記憶力真好 |
| word | 這個字你以前就認得吧 | 對啦，這個字你小時候就會了 | 今天還認這麼多字，真棒 |
| differences | 看看這兩排哪一個不一樣？ | 你眼睛真利！ | 你觀察力比我還好 |
| clock | 以前上學都要看這個呢 | 你看時間很準 | 看時鐘不難吧？ |
| matching | 看看哪個配哪個，慢慢來 | 對！就是這一對 | 都配對完了，好厲害 |
| shape | 看看是什麼形狀 | 對啦，一樣的形狀 | 這些形狀你都認得 |
| picture | 這個你認得吧，以前家裡可能有 | 沒錯！ | 這些東西你都認得好熟 |
| number | 以前買菜都要算錢的 | 對，就是這個數字 | 數字你還是那麼清楚 |
| sorting | 從小到大排排看 | 對！你排得真好 | 你很會排喔 |
| classify | 想想這個是哪一類 | 沒錯，就是那邊 | 分類對你來說不難吧 |

### 邊界情況

- 如果玩家在 5 秒內連續答對多次，banner 會被重複觸發——使用 debounce 或只在「狀態從非答對變成答對」時觸發。
- 遊戲從 💬 展開回 banner 時，若沒有對應的時機（例如剛進入還沒答對），顯示 start 提示。
- 點擊 💬 展開後，若 5 秒內沒有新事件，一樣 fade 回 💬。

## 功能 2：今日摘要

### 目的
照護者用完要離開前，一鍵產生「可以貼到家族 LINE 群」的文字紀錄。

### 觸發條件
- 首頁 `.difficulty-pill` 下方新增一個按鈕「📝 把今天傳給家人」
- 按鈕**只在當次 session 至少完成 1 個活動後才出現**（完成數 `>= 1` 才 render）
- 頁面 reload → session state 歸零 → 按鈕消失

### Session 狀態

```js
APP.session = {
  startTime: Date.now(),        // 第一次打開首頁的時間
  completed: [],                // [{ id, name, icon, finishedAt }]
};
```

- 只記錄**完成到 complete screen** 的活動（中途離開不算）
- 任何遊戲跑完 → push 一筆進 `completed`
- 不存 localStorage，純 JS 記憶體

### 首次稱呼設定

- 第一次點「把今天傳給家人」時，先跳小 modal：
  ```
  請問您要怎麼稱呼長輩？
  
  〔爸〕 〔媽〕 〔奶奶〕 〔外公〕 〔其他：____〕
  
  （之後都會這樣寫，可以隨時改）
  ```
- 選完後存 `localStorage.elderName`
- 日後可以從難度 modal 裡的「設定」入口重新改（或先不做 UI，使用者清 localStorage 就會重問）
- 如果使用者關閉 modal 沒選 → 預設用「家人」

### 摘要 modal

按下「把今天傳給家人」後開啟一個預覽 modal：

```
┌──────────────────────────────┐
│   今天的紀錄                    │
├──────────────────────────────┤
│                              │
│   📝 2026/4/20 下午            │
│   今天和爸爸一起玩了 8 分鐘       │
│                              │
│   ▸ 呼吸練習                   │
│   ▸ 顏色辨識                   │
│   ▸ 翻牌配對                   │
│                              │
│   他配合度很好，狀態平穩 😊     │
│                              │
├──────────────────────────────┤
│   〔📋 複製傳 LINE〕  〔我不要傳〕│
└──────────────────────────────┘
```

### 文字格式（完全固定）

```
📝 {日期字串} {時段}
今天和{稱呼}一起玩了 {X} 分鐘

▸ {活動 1}
▸ {活動 2}
▸ {活動 3}

{描述句}
```

**各欄位產生規則：**

| 欄位 | 規則 |
|------|------|
| 日期字串 | `YYYY/M/D`（例 `2026/4/20`） |
| 時段 | 依 startTime 小時判斷：0–5 凌晨 / 5–11 早上 / 11–13 中午 / 13–18 下午 / 18–22 晚上 / 22–24 深夜 |
| 稱呼 | `localStorage.elderName` 或預設「家人」 |
| X 分鐘 | `ceil((now - startTime) / 60000)`，最少顯示 1 分鐘 |
| 活動列表 | `session.completed` 依完成順序，取遊戲中文名 |
| 描述句 | 依完成數（見下表） |

**描述句邏輯：**

| 完成數 | 描述句 |
|-------|--------|
| 1 | 一起做了一件事，陪著他就很好 |
| 2 | 他配合度不錯，狀態還可以 |
| 3 | 他配合度很好，狀態平穩 😊 |
| 4–5 | 今天狀態很好，玩了好幾樣 🌟 |
| 6+ | 今天狀態非常好，玩了一圈呢 🎉 |

### 複製行為

- 點「📋 複製傳 LINE」→ `navigator.clipboard.writeText(summary)` 
- 成功 → toast「已複製，貼到 LINE 群就好」，1.5 秒後 modal 關閉
- 失敗（舊瀏覽器、或 clipboard permission denied）→ 
  - 顯示一個 readonly `<textarea>` 裡面是 summary 文字，已 pre-select，讓使用者 Ctrl+C 手動複製
  - 下方文字：「複製不了，請長按選取後複製」

### 邊界情況

- 使用者完成了活動但沒寫稱呼就按 cancel → 按鈕回到原樣，session state 不動，下次點會再問一次
- 使用者完成活動後又重新進入該活動再完成一次 → 同一個活動重複出現在列表中（OK，代表真的玩兩次）
- 使用者在 modal 預覽時又偷偷離開去玩遊戲 → 關掉 modal 即可，回首頁按鈕仍然可按（會顯示最新資訊）
- session 時間跨日（例如晚上 11:30 打開，半夜 0:30 還在玩）→ 日期取第一次打開的 startTime 的日期，時段描述同理

## UI 影響範圍

### 新增 DOM

- 每個遊戲 screen 內：頂部 `.tip-banner` + 右上 `.tip-icon`
- 首頁 `#home`：`.difficulty-pill` 下方多一個 `.summary-btn`（條件顯示）
- 新 modal：`#summary-modal`
- 新 modal：`#elder-name-modal`
- `.tip-banner` 相關 class 與動畫

### 不改動的既有行為

- 推薦式首頁邏輯
- 難度 modal
- 遊戲本身玩法與計分
- localStorage 既有的 `dementiaAppV2`（會在旁邊新增 `elderName`，不合併）

## 資料持久化清單

| Key | 用途 | Scope |
|-----|------|-------|
| `dementiaAppV2` | 既有：難度、最後玩的時間 | 永久 |
| `elderName` | 新：長輩稱呼（一個字串） | 永久 |
| `APP.session.startTime` | 新：當次 session 開始時間 | 記憶體，reload 歸零 |
| `APP.session.completed` | 新：當次完成的活動列表 | 記憶體，reload 歸零 |

## 驗收條件

- [ ] 進入任一遊戲，頂部 5 秒內看到該遊戲的 start 提示
- [ ] 提示 fade 後右上角有 💬 icon，點了能再展開
- [ ] 答對時 banner 切換到 correct 提示
- [ ] 完成畫面顯示 complete 提示
- [ ] 回首頁沒有紀錄 → 沒有「把今天傳給家人」按鈕
- [ ] 完成 1 個活動後回首頁 → 出現按鈕
- [ ] 點按鈕第一次 → 先問稱呼，填完才看到摘要
- [ ] 摘要 modal 顯示正確的日期、時段、稱呼、分鐘數、活動列表、描述句
- [ ] 完成數對應的描述句正確切換
- [ ] 點「複製傳 LINE」成功複製，並顯示 toast
- [ ] Reload 頁面後，按鈕消失，session 歸零
- [ ] `localStorage.elderName` 還在，不會被清掉
- [ ] 視覺對比達 AAA 7:1（banner 深灰底白字需符合）
- [ ] 15 個遊戲各自的 3 條提示語都對應寫進 TIPS 資料結構
