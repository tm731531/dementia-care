# 陪伴小幫手 v2（試驗版）— 推薦式首頁重構

> v2 不是把舊版砍掉，是在同一個 repo 裡開第二個版本**實驗不同的設計方向**。
> 核心轉換：從「遊戲選單，10 格自己挑」→「今天一起做這個好嗎？一鍵開始」。
> 附加：給照護者的陪伴指南跟每日摘要分享，讓陪伴變得更容易。

純前端、單檔 HTML、零廣告、完全離線可用。**跟 v1 並存，使用者可以對照比較**。

---

## 快速開始

1. 打開 `index.html`（或 <https://tm731531.github.io/dementia-care/dementia-companion-v2/>）
2. 首頁直接看到「今天來玩這個好嗎？」+ 一個推薦的活動
3. 按「好，開始 →」直接進入，或按「換一個」再推薦，或「我想自己選」看完整清單
4. 每個遊戲開始時頂部會有一條💬 給照護者的提示，5 秒後收成小 icon 可再展開

不需要帳號、不需要安裝、不需要網路。設定跟歷史存在瀏覽器 localStorage（key：`dementiaAppV2`），跟 v1 完全獨立。

---

## 為什麼做 v2

跟 v1 完全一樣的需求（失智照護遊戲），但我發現 v1 首頁的「10 格遊戲讓照護者挑」其實把**決策成本**又丟回給照護者。這個 APP 的目標使用者是**內向、疲累、不擅長做選擇的照護者**——他來找工具不是要再做一次選擇。

**「今天來玩這個好嗎？」** 這句話把選擇從使用者身上拿走：
- 依時段（早/午/晚）挑適合的活動類型
- 依上次玩過的日期避免重複
- 照護者點一下就開始，不用自己想

**次要選項（換一個 / 我想自己選）** 字體縮小、沒底色、不搶 C 位，降低選擇癱瘓。

## 跟 v1 的差異

| 項目 | v1（`dementia-companion/`） | v2（`dementia-companion-v2/`） |
|------|------|------|
| 首頁 | 10 格遊戲卡讓使用者選 | 1 張「今天推薦這個」卡 |
| 難度設定 | 頂部滑桿常駐 | 收進 modal，點「調整」才出現 |
| 遊戲數量 | 15 個認知遊戲 | **15 個全部移植過來** |
| 陪伴指南 | 無 | 每個遊戲 3 條 × 15 遊戲 = **45 條可直接對長輩說的句子** |
| 每日摘要 | 無 | 完成活動後一鍵複製可貼 LINE 的今日紀錄 |
| localStorage key | `dementiaApp` | `dementiaAppV2`（獨立，不互相干擾） |

**v1 沒被替換掉**——首頁 `index.html` 同時列出 v1 跟 v2，你想要哪個都可以點進去用。

## 照護者支援功能（v2 才有）

### 💬 陪伴指南（每個遊戲 3 條可直接說的話）

進入遊戲的 5 秒內，頂部會出現一條深灰 banner：

```
💬 哇，好多顏色，你以前最愛穿什麼顏色？
```

5 秒後 banner 摺疊成右上角的 💬 icon，點一下可以再展開。答對時會換成「答對時可以說的話」，完成時會換成「結束時可以說的話」——15 個遊戲 × 3 時機 = **45 條** 全部照一個可預期的格式：`情境 → 可以直接說的話`。

這個提示是**給照護者讀的**，**不會被唸出來**（那是長輩的語音指令）。照護者用自己的聲音、自己的語氣說才有溫度。

### 📝 今日摘要（分享給家人）

每完成一個活動，session 會記錄一筆。首頁下方會出現「把今天傳給家人」按鈕（完成 ≥ 1 個才出現）。按下去會預覽可以貼到家族 LINE 群的文字：

```
📝 2026/4/20 下午
今天和爸爸一起玩了 8 分鐘

▸ 呼吸練習
▸ 顏色辨識
▸ 翻牌配對

他配合度很好，狀態平穩 😊
```

**預覽後才複製**，照護者確認內容才送出。格式永遠固定，多次使用後照護者學一次就預期得到。

**唯一用到的 localStorage key 是 `elderName`**（你稱呼長輩的方式：爸、媽、阿嬤⋯），不做資料庫、不存活動歷史，reload 就歸零。

---

## 核心設計原則：可預期

整個 v2 圍繞這個原則：

- 提示永遠在**同一位置、同一模板**（`情境 → 直接說的話`）
- 摘要格式每次**完全一致**，分享前有預覽
- 照護者**學一次就預期得到**，不用每次重新解讀 UI

失智照護者的腦力已經被陪伴消耗很多了，工具不該再增加認知負擔。

## 故事

這個 v2 是用 [Claude Design 流程](https://blog.tomting.com/2026/04/21/claude-design-brainstorm-spec-plan-workflow/)（brainstorming → spec → plan → subagent-driven-development）做的。過程中學到的事：
- 規則化 SDD 流程救了重構的前半段（砍功能、寫一致的 45 條提示）
- subagent-driven-development 對單檔 HTML 原型**過殺**，後半段改回 inline
- 最關鍵的 bug（語音堆疊、色差、身體部位遊戲退化）都是**朋友實測後抓出來的**，不是流程跑出來的

這個 repo 的 `dementia-companion-v2/` 本身就是那篇部落格的實戰場域。
